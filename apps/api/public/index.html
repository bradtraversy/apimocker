Compact API reference

The routes you need to start building.

Base URL: https://apimocker.com

Core CRUD resources

All four resources support create, read, full update, partial update, and delete operations.

GET POST PUT PATCH DELETE
/users /posts /todos /comments

Per-resource search

Use a required, case-insensitive q value.

  • /users/search?q=john
  • /posts/search?q=development
  • /todos/search?q=review
  • /comments/search?q=great

Relationships and likes

  • GET /users/:id/posts
  • GET /users/:id/todos
  • GET /posts/:id/likes
  • POST /posts/:id/likes

Likes accept an optional userId. Omit it for an anonymous like.

Latency, errors, and health

  • GET /posts?_delay=2000
  • GET /error/404
  • GET /error/500
  • GET /error/validation
  • GET /error/timeout
  • GET /health

Response shapes

Lists: an object with a data array and a pagination object.

Single and write: the resource object directly, without a data wrapper.

Delete: 204 No Content with an empty body.
